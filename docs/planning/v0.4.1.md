# v0.4.1 — Operational Polish: Release Plan

**Theme:** Fix what broke during v0.4.0 operation, stabilize what shipped, unblock consumer adoption.  
**Status:** Active — in progress  
**Milestone:** [v0.4.1 — Operational Polish](https://github.com/thurlow-research/HumanOversightSystem/milestone/7)  
**Cuts after:** All blocking/severe issues resolved and CPS successfully running on v0.4.0.

---

## Theme rationale

v0.4.0 shipped a working autonomous loop. v0.4.1 is the patch that cleans up what the ship uncovered: auth infrastructure bugs found during release cut validation, consumer install gaps found during CPS upgrade testing, and validation infrastructure non-determinism that blocked the release cut itself. Scope is strictly blocking/severe — everything else waits for v0.5.0.

**Triage gate:** Would this prevent a consumer from running HOS, cause data loss, or create a security vulnerability? If yes → v0.4.1. If no → v0.5.0+.
