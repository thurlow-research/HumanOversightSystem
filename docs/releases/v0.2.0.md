# v0.2.0 — Orchestration, auditable upgrades, and the triage discipline

A substantial minor release. The theme: **make the oversight pipeline get *used* (not bypassed), make upgrades *auditable*, and make findings *triaged* (not chased).** Most of this is consumer-facing — if you're on v0.1.2, upgrading is worth it.

> **Heads-up (one behavior change):** the framework now establishes an **orchestrator role** and auto-loads it into your `CLAUDE.md`. See *Upgrading* below. The machine-account identity model (`AGENT-IDENTITY.md`) ships as a **spec only** in this release — the implementation lands in v0.3.0.

---

## ⭐ Highlights

- **Orchestrator role — stop the pipeline-bypass (#172).** The single most important fix. The human-facing agent was doing validators/reviews/code *itself* instead of dispatching the specialized agents — so the pipeline silently went unused. The new **"Orchestrate, Don't Absorb"** principle (in `AGENTS.md`) is now **auto-loaded into your `CLAUDE.md`** by the installer, so the agent knows to delegate by default. If you saw your agent "just doing the work," this is the fix.
- **`HANDLING-FINDINGS.md` — stop chasing false positives (#158).** Validators are *signal*, not gates; they over-flag by design. This guide teaches the triage discipline — **fix / accept-with-rationale / file `scanner-fp` upstream** — so you stop refactoring good code to silence a heuristic.
- **Auditable, reversible upgrades (opt-in) (#193/#194).** `hos_install.sh --pr` applies an upgrade on a branch and opens a PR — reviewable diff, revert to roll back, `main` untouched until you merge. Opt-in this release; default in a later one once it's proven.
- **Obsolete-file detection + safe prune (#182).** The installer now tracks a `.hos-manifest` and **warns** when a framework file was removed across versions but lingers in your repo (a stale `.claude/agents/*.md` is a real AI-confusion risk). `--prune` (opt-in) **archives** them — move, not delete — to a committed, quarantined `.hos-archive/`, and only if unmodified. Recoverable three ways.

## 🔧 Installer & upgrades
- One run produces a fully-configured project — `hos_install.sh` now delegates interactive config-gen to the config tool (#87), and the legacy `setup_oversight.sh` is retired.
- **Fix:** the installer no longer copies its source `.venv`/`__pycache__` into your repo (#178).
- New flags: `--pr` (auditable upgrade PR, opt-in), `--prune` (archive obsolete files, opt-in).

## 🧭 Oversight & methodology
- **Reliability/ops reviewers are now force-required at HIGH+ when the diff warrants it (#135)** — closes the asymmetry with security/privacy (conditional on the diff, not blanket).
- **Convergence by disposition (#133)** — a first-class triage/accept step: convergence = every finding *dispositioned* (fix / file / accept-residual / noise), not every finding *fixed*, with an anti-gaming guardrail (the AI may not silently downgrade a real finding to "accept").
- **Human-gate pilot accountability (#127)** — a committed, content-hashed authorization record makes a self-authorization detectable after the fact.

## 📋 New conventions & docs
- **Cross-repo conduct (#188)** — how agents behave in repos they don't own (guests don't merge; our-bug → upstream + `upgrade-hos` tag; their-fix → PR for *their* review; non-repro → detail + flag).
- **Issue-clarity standard (#186/#195)** — every human-review issue assumes zero reader context: *What it is / Impact / Options with pros+cons / Recommendation* (labeled separately).
- **`AGENT-IDENTITY.md` (spec)** — the two-account (worker/overseer) machine-identity model for v0.3.0.

## 🐛 Fixes
- `run_validators.sh` no longer collapses to a false CRITICAL when files arrive as one whitespace-joined argument (#149).
- `scripts/oversight/*.py` now pass the isort/black gates they ship (#162); `ops-reviewer.md` doc-path consistency (#179); `DECISIONS.md` D40 corrects the missing-risk-assessment handling (#180).
- New **collection-integrity gate** catches a deletion that orphans tests elsewhere (#157).

## ⬆️ Upgrading from v0.1.2
```bash
# from your bootstrap/ folder (or a clone's bootstrap/):
./hos_install.sh --force /path/to/your-project        # in-place (default)
#   or, to review the upgrade as a PR first:
./hos_install.sh --pr --force /path/to/your-project
```
After upgrading, **check your `CLAUDE.md`** — the installer adds/refreshes an HOS orchestrator block (between markers; your own content is untouched). If the installer warns about obsolete framework files, review them and optionally `--prune` (archives, doesn't delete).

> **Erratum (added in v0.2.1).** The "did run and did its job" above refers to the **release gate** (`validate_agents.sh`), which already carried JSON-salvage and caught the real contradictions. It does **not** describe the **consumer-facing per-step** cross-vendor review (`run_second_review.sh`), which in v0.2.0 was **non-functional** — codex was invoked with the dead `--quiet` flag and agy's prose was discarded, so a MEDIUM+ second-review pass *looked* clean while the reviewers were never actually invoked.
