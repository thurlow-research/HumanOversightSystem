**Theme:** Close the remaining governance gaps from v0.4.x field operation; make the audit trail structurally conflict-free; harden the fail-closed posture across the entire pipeline.

71 issues closed. No open issues in the milestone at cut.

## Headline: Audit log is structurally conflict-free

The audit trail is no longer a single append-only JSONL file. It is now a directory of immutable, content-addressed, per-entry JSON files under `audit/log/<YYYY>/<MM>/`. Two branches writing audit events in parallel can now both merge to `main` with zero conflicts — not because of a workaround, but because distinct filenames cannot conflict.

- **Per-entry helpers** — canonical writer (`audit_log.py` / `audit_log.sh`) and read-shim; all existing readers produce identical output (#888 P1, #929)
- **All writers migrated** — `cycle_log.py`, `suspension_manager.py`, `run_with_retry.sh`, `release_artifact_logic.py` (#888 P2, #930)
- **All readers migrated** — `step_range.sh`, `audit_conditional_proceed.sh`, `cycle_log.py` (#888 P3, #931)
- **Historical migration** — legacy `oversight-log.jsonl` converted, round-trip clean (#888 P4, #932)
- **Sync machinery retired** — `.gitignore` entries removed, `sync-audit-log.yml` deleted, `audit/oversight-log.jsonl` untracked; per-entry files commit inline with their PR (#888 P5, #947)

Root cause shared with: #873, #849, #842, #832, #821, #820, #813, #809.

## Fail-closed hardening

Thirteen fail-open bugs closed — every one now fails closed or errors out:

- **`run_red_team.sh` milestone checkpoint** exits 0 with zero reviewers — now fails closed (#911)
- **Halt kill-switch** fails open on `gh` API error or missing repo slug — now fails closed (#912)
- **`audit_conditional_proceed.sh`** conflates API error with `NO_MERGE` — now emits `UNKNOWN` (#913)
- **`hos_install.sh --prune`** silently implied `--squash`, overwriting consumer-owned PROJECT regions — now independent flags (#914)
- **Overseer open-PR fetch error** triggered a 30-min backoff instead of a retry — now retries cleanly (#915)
- **`hos-worker-cron`** lost Claude's exit code under `set -e` — overseer now receives the correct handoff signal (#916)
- **`static_analysis` + `migration_scorer` validators** failed open when tool was absent or output unparseable — now fail safe with scored signal (#917)
- **`run_second_review.sh` reviewer selection** failed open on `SELECT_OUT` eval error — now errors out (#681)
- **`run_panel.sh` required reviewers** could skip/exit early and still converge clean — now blocked (#682)
- **Validation-logic convergence** could launder `request_changes` into approval — fixed (#683)
- **`--risk` override** bypassed the deterministic triage floor in `run_panel.sh` — now clamped (#910)
- **Bot identity guard** (`HOS_EXPECTED_BOT_LOGIN` vs `HOS_BOT_LOGIN`) was a tautology — now non-tautological (#703)
- **Duplicate audit PRs** — overseer created a new audit PR per cycle for the same blocked PR — now idempotency-checked (#849)

## Governance

- **Content-hash stamps** — validation stamps are now anchored to a SHA256 of the validated agent files; eliminates the merge-conflict cascade that blocked 15+ PRs in v0.4.x (#552)
- **SHA-256 tarball verification** — `hos_install.sh` now fetches and verifies `SHA256SUMS` before extracting the release tarball (#684)
- **Priority-based issue selection** — worker and `hos-cron` context block now order candidates by `priority:critical > high > medium > low`, then issue number; a `next_candidates.jq` filter is the single source of truth for both paths (#901)
- **Scripts-review dedup ledger persistence** — the LLM reviewer dedup ledger is committed in-repo, eliminating non-determinism across cycles (#686)
- **Worker stacked-branch fix** — worker no longer includes in-flight commits from a prior branch when building a new PR (#850)

## Performance

Four cron-launcher optimizations carried over from the v0.4.2 cycle:

- **Skip Claude launch** when all open worker PRs are awaiting human merge — nothing to do (#791)
- **Pre-compute cycle context** (open PRs + next-work candidates) before Claude starts — no API calls needed inside the turn (#792)
- **Halt check before Claude launch** — exits immediately if a `hos-halt` issue is open (#793)
- **Agent-file availability check** before Claude launch — avoids a session that fails immediately (#794)
- **Skip inner-loop tests** when HEAD is known-clean (cowpat match) — removes redundant test runs from idle cycles (#789)

## `hos-suspend` CLI

`bin/hos-suspend <project>` pauses a project's autonomous work without killing cron. Leaves cron running and the loop intact; suppresses worker cycles until unsuspended (#778).

## Upgrade notes

- **Audit log**: `audit/log/` is now the authoritative record. `audit/oversight-log.jsonl` is untracked and can be archived or deleted. The read-shim (`scripts/oversight/lib/audit_log.py` or `audit_log.sh`) globs the directory and emits the existing JSONL stream — readers need no changes.
- **Run the migration script**: `scripts/migrate_audit_log_to_dir.sh --log <path/to/oversight-log.jsonl> --root <repo-root>` to backfill any consumer deployment's existing audit history. Script is idempotent and verifies round-trip integrity. **Delete the script after both HOS and CPS complete the upgrade** (it is explicitly throwaway per #888 R7).
- **Stamp redesign**: validation stamps are now content-hash filenames under `audit/validation/`. The `.gitignore` exemption and the gitignore-as-conflict-mitigation pattern are retired.
- **`hos_install.sh`**: re-run to pick up content-hash stamps and the updated ship-set. Pass `--pack <name>` to re-inject stack-depth regions.
