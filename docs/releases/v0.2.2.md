# v0.2.2 — Install == manifest (no agent drift)

A focused patch. Fixes the install/manifest drift CPS surfaced upgrading to v0.2.1 (#225).

## The fix
`hos_install.sh` had drifted: the agent copy-loop installed a hardcoded **6** agents while the `.hos-manifest` enumerator did `find .claude/agents` over **all 16** in the source. As `ops-*`, `reliability-reviewer`, `post-change-sweep`, etc. were added, the manifest auto-tracked them but the install never shipped them — so consumers got a **superset manifest** *and were missing agents their own pipeline dispatches* (`risk-assessor`→`prompt-fidelity`, `ops-reviewer`, `reliability-reviewer`, `post-change-sweep`).

**Now:** `scripts/framework/consumer_agents.txt` is the single source of truth, read by **both** the copy-loop and the manifest enumerator — so manifest == install by construction (#225).

- **Ships (12 consumer oversight agents):** risk-assessor, dep-mapper, risk-historian, oversight-evaluator, oversight-orchestrator, spec-red-team, prompt-fidelity, ops-designer, ops-reviewer, reliability-reviewer, post-change-sweep, ux-designer.
- **Excluded (4 framework-dev validators — they validate HOS itself):** framework-validator, framework-setup-validator, doc-validator, spec-compliance-validator.

## ⬆️ Upgrading from v0.2.1
```bash
./hos_install.sh --pr --force /path/to/your-project    # review the upgrade as a PR
```
The upgrade installs any previously-missing oversight agents and produces a clean `.hos-manifest` (every tracked file present). Your own base/worker agents (coder, reviewers, …) are untouched.

> **Note:** base/worker agents remain consumer-owned in this release. HOS shipping a canonical, upgradable base team (core + reusable stack packs) is designed for **v0.3.0** alongside machine accounts.

Verified by clean-install and CPS-upgrade tests: 12 agents installed, all manifested, 0 missing.
