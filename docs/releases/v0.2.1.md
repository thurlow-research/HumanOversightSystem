# v0.2.1 — The cross-vendor review actually works now

A focused patch release. The theme: **make the cross-vendor second review functional**, because in v0.2.0 it silently wasn't — and that's the gate consumers rely on for every MEDIUM+ change. If you run HOS on a real project, upgrading is worth it.

> **For consumers (e.g. CPS):** this is the release to upgrade to. v0.2.0's `run_second_review.sh` would *look* like it ran agy/codex and found nothing, while actually never invoking them. After v0.2.1 a non-review is a non-review — loud, not silent.

---

## ⭐ Highlights

- **Cross-vendor second review keystone (#201).** Two independent bugs made `run_second_review.sh` produce a clean-looking "no findings" pass without ever running the reviewers:
  - **codex (#199):** it was invoked with the long-removed `codex --quiet` flag; the call always failed and the error was swallowed into an empty `verdict:error`. Now uses `codex exec`.
  - **agy (#113):** agy has no JSON-output mode and intermittently returns prose narration; the old path discarded it as an empty error. Now the harness **salvages** the JSON out of any prose/markdown wrapper, **retries once** with a JSON-only reinforcement, and falls through to the existing `parse_prose` path so a real prose review is preserved as `unparseable` (a human reads it) — **never** silently dropped or silently passed.
- **Honest degradation, end to end.** A reviewer that did not actually review now surfaces a *distinct, loud* signal. `oversight-evaluator` already distinguished `error` (runtime crash → re-run) from `unparseable` (review exists, isn't machine-structured → human reads it); v0.2.1 makes the script actually emit those states instead of collapsing everything to a masked empty pass.
- **Risk-assessment scope-pin + blocking-finding enforcement (#204).** `risk-assessor` now scopes to the step's `BASE_SHA..HEAD_SHA` commit range — **not** the post-commit-empty `git diff HEAD` — so it can't emit a valid-looking assessment for an empty/partial file set. It writes a machine-readable `base_sha`/`head_sha`/`files_assessed`/`blocking_findings` header, and `oversight-evaluator` Phase-1 **fails** on a scope mismatch or any unresolved blocking finding (e.g. a non-suspended dep-mapper `Data confidence: LOW` at HIGH+ — which previously had no consumer and couldn't stop the PR).

## 🔧 Tooling reliability
