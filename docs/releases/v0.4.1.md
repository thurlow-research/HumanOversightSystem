# v0.4.1 ‚Äî Operational Polish

**Theme:** make the autonomous worker‚Üîoverseer loop *real and proven* ‚Äî and harden everything the v0.4.0 release-cut validation surfaced.

This release turns the autonomous pipeline from "runs the loop" into "runs unattended on a schedule, on the subscription, with the overseer merging its own approved PRs." It also closes the large batch of fail-open and portability bugs found during the v0.4.0 cut, and lands the install path so consumer projects get the cron agents.

---

## Headline: the autonomous cron loop is whole

The worker and overseer now run as **unattended OS-cron agents** and complete the full loop end-to-end ‚Äî including the overseer **auto-merging** its own approved, non-protected PRs (verified live on this release: #742/#743 merged by `hos-overseer-hos[bot]`, no human bypass). Protected/control-defining changes still require a human (dual-control), exactly as designed.

- **`bin/hos-cron`** ‚Äî one parameterized launcher for every cron agent (`--role worker|overseer --project <name>`), reading a machine-local project registry (#713).
- **Headless subscription auth** ‚Äî `claude --print` runs on the user's Claude subscription via a long-lived OAuth token, not API credits; the launcher pins its own environment so a bare crontab line works (#728).
- **Headless tool permissions** ‚Äî cron sessions clear the approval gate (`--permission-mode bypassPermissions`, no human present to approve), with a documented layered security model: human `needs-ai` triage gate, OAuth-token isolation from tool subprocesses, short-lived scoped GitHub token, and prompt-injection hardening (`docs/CRON-SETUP.md ¬ß8`).
- **File-based wakeup + adaptive backoff** between worker and overseer ‚Äî fast handoff when there's work, near-zero cost when idle (#628).
- **`bin/hos-trim-logs`** ‚Äî weekly cron-log rotation (#714).
- **Auto-merge unblocked** ‚Äî branch-protection required status-check contexts now match the workflow job names that actually post; previously every overseer merge returned HTTP 405 while the gates ran green (#737). A committed `setup_branch_protection.sh` + `test_branch_protection_contexts.py` make the configuration reproducible and drift-proof.

## Consumer install of the cron agents

- `hos_install.sh` installs `bin/hos-cron` + `bin/hos-trim-logs` and generates project-customized cron-prompt files; "Next steps" prints ready-to-paste crontab entries (#715, #717).
- Deterministic git credentials in the launcher ‚Äî no longer depends on whatever credential helper the host has (#738).
- Fixed a developer-home-path hardcode in the cron-prompt substitution (#731).
- Full macOS **and Ubuntu** setup guide: `docs/CRON-SETUP.md`.

## Auth & identity hardening

- Consumer-configurable config path (`HOS_CONFIG_DIR`) and identity guard no longer hardcodes HOS bot names ‚Äî unblocks all consumer projects (#696, #697).
- Identity guard now verified against the GitHub-API-derived bot identity, closing a circular `apps.env` dependency; JWT/token handling tightened (#631, #632).

## Oversight correctness ‚Äî fail-open ‚Üí fail-closed

A cluster of gates that silently *passed* on error now *block*:
- Overseer reads the committed validator artifact, not just CI status, before a merge decision (#555).
- `validate_scripts.sh` / `validation_logic.py` treat a hung or erroring reviewer as **blocking**, not a silent approve (#669, #670).
- `security_scan.sh` treats pip-audit "vulnerabilities found" as a fail, not an execution error (#672).
- `collection_integrity.sh` compares against the merge base, so committed PR changes are actually checked (#673).
- `hos_worker.sh` budget/failure gates fail closed (#678).
- `machine_lock.sh` release verifies ownership before removing the lock (#677).
- PR size limits enforced (‚â§15 files / ‚â§10 commits; 25-file hard ceiling) (#554).

## Known limitations (tracked, not regressions)

- **Scripts-review gate non-determinism.** The release-cut validation's adversarial LLM reviewers don't converge on a large script corpus (#686); this cut may require `-- --skip-codex` to drop the non-deterministic adversary, as prior cuts did. The durable fix ‚Äî a severity√ócorroboration convergence matrix ‚Äî is scheduled for **v0.5.0** (feasibility backtest first: #740 ‚Üí #686).
- **`bin/` is not yet a protected surface** (#744, v0.4.2) ‚Äî the cron launcher is control-defining (sets `bypassPermissions`, handles tokens) but currently auto-merges. Until then, treat launcher changes with extra care.
- **Overseer cannot yet *execute* the merge on human-approved protected PRs** (#741, v0.4.2). Protected-surface merges currently require the human to both approve *and* merge; the dual-control "human authorizes, overseer executes" model lands in v0.4.2.
